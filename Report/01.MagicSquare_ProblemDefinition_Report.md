## 1. Executive Summary

4×4 부분 마방진(빈칸 2개, 합 34)을 **손으로** 푸는 학습자는, **행 합 34만**으로 빈칸 후보를 좁히다 **열·대각선은 넣어 본 뒤에야** 확인하고, **왜 틀렸는지 보기보다 맞을 때까지 바꿔 넣는** 방식으로 **약 25분**(후보군 찾기 **약 24분**)을 쓴다.

본 프로젝트는 솔루션(앱·Solver·UI) 선행 없이, **채워진 격자가 10축(행 4·열 4·대각 2) 합=34를 만족하는지** 판정하는 **Rule · Command · Test Loop**를 먼저 고정하여, “행만 맞으면 된다”는 착각과 삽입 후 깨지는 열·대각을 **재현 가능한 실패 조건**으로 다룬다.

## 4. Mom Test — 과거 사실 (증거)

### 4.1 인터뷰 답변 (2차 — SSOT)

| # | 과거 사실 |
|---|-----------|
| 1 | **올해 1월**, 슬라이드와 같은 4×4·빈칸 2개·합 34 문제 **제출 완료** |
| 2 | **시작~제출 약 25분** |
| 3 | 가장 오래 걸린 구간: **빈칸 2개 숫자 정하기** |
| 4 | **후보군 찾기 약 24분**, **하나씩 넣어보기 약 30초** |
| 5 | 후보 좁히기: **행 합 34 우선** |
| 6 | **열·대각선은 넣어 본 뒤** 확인 |
| 7 | 하나 **반대로** 넣어 34 불일치 → **빈칸 숫자 바꿔넣기** (약 **5번**, **맞을 때까지**) |
| 8 | 후보군 **종이에 적으며**; **중복 숫자 미사용** |
| 9 | 제출 직전 **대각선부터** 검산 (평소 어려운 것 우선) |
| 10 | **손으로만**; 1월 이후 같은 문제 **없음**; **코드 없음** |
| 11 | 가장 답답: **빈칸마다 하나씩 넣어봐야 함** |
| 12 | 행 합 34로 후보 **1개**만 찾음 (빈칸당·조합 **미확인**) |

### 4.2 Mom Test 증거 3줄 (직접 인용)

1. “**행 합 34 우선** 기준을 잡았어” / “**넣어본 뒤에**” (열·대각선 확인)
2. “**맞을 때까지 바꿔본 것**이지, 안 되는 것을 먼저 찾기보다는” / “**5번** 정도 시도”
3. “**후보군 찾기**에 시간을 보냈지” / “**적어내며** 했었지” / “후보는 **1개밖에 못 찾았어**”

### 4.3 1차 Mom Test (보조 — D-002 보강)

| 항목 | 1차 (참고) | 2차 (SSOT) |

## 5. 문제 정의

### 5.1 표면 문제 (잘못된 정의 — 하지 않을 것)

- “4×4 마방진 **검증 프로그램**을 만든다”
- “ECB로 **SquareValidator**를 만든다” (문제 정의로서)
- “대각선을 **자동으로 체크하는 앱**을 만든다”
- “**Solver**로 빈칸 2개 **자동 채우기**”
- **후보군·종이 계산** 자동화 (24분 구간 — Session 3 범위 밖)
- **행만** 검증하는 판정 로직

### 5.2 진짜 문제 (한 문장)

> 손으로 4×4 부분 마방진을 풀 때 **행 합 34만**으로 빈칸 후보를 종이에 적어 좁히는 데 **약 24분**을 쓰고, **열·대각선은 넣어 본 뒤에야** 확인하며 **왜 틀렸는지 보기보다** 빈칸 숫자를 **맞을 때까지 바꿔 넣는** 방식으로 **약 25분**을 쓴다.

**포함 요소**

| 유형 | 내용 |
|------|------|
| 불편 | 하나씩 넣어보기 · 바꿔넣기 · 맞을 때까지 시행착오 |
| 비용 | 약 25분 (후보 24분 + 삽입 30초) |
| 판정 | 행 합 우선 후보 → 삽입 후 열·대각 확인 |
| 재현 | 5회 바꿔넣기; 원인 분석 없이 맞출 때까지 |

## 6. 문제 → 해결 방향 (솔루션 최소화)

| Mom Test 문제 | 1차 대응 (Session 3) |
|---------------|----------------------|
| 행 합만 보면 “됐다” 착각 | **10축 합=34** Rule(R1) + `validate_lines` |
| 넣은 뒤 열·대각 깨짐 (“반대로 넣었더니”) | D-002: 행=34·열/대각 fail + `failed_lines` |
| 맞을 때까지 5번 시도 | D-002로 **최종 잘못된 격자**를 fail로 고정 |
| 후보 24분·종이 탐색 | Session 3 **범위 밖** — 삽입 **후** 판정만 SSOT |
| 제출 전 대각선 우선 | `failed_lines`에 **D1/D2** 식별 (Skill 체크리스트) |

**주제 한 문장 (Session 3)**

> 4×4 부분 마방진을 채운 뒤, “행만 맞으면 된다”고 느끼는 순간과 실제로 열·대각선이 깨지는 순간을 구분할 수 있게, 채워진 격자가 마방진 조건을 만족하는지 판정하는 기준을 먼저 정한다.

## 11. 결론

MagicSquare_1004의 문제는 “마방진 앱·Solver 부재”가 아니라, **행 우선 판단과 삽입 후에야 드러나는 열·대각 불일치**를 **체계적으로 걸러내지 못하고**, **맞을 때까지 시행착오**로 시간을 쓰는 것이다.

Session 3는 Solver·UI·후보 탐색 이전에 **`validate_lines` + Rule + Test Loop**로 Mom Test 2차 증거를 **재현 가능한 실패 조건(D-001~D-003)** 으로 고정한다.
