## Mom Test 결과 (입력)

### 페르소나

4×4 부분 마방진(빈칸 2개, 합 34)을 **손으로** 푸는 학습자. **코드로 다룬 적 없음**. 어려운 것(대각선 등)부터 처리하는 습관 있음.

### 진짜 문제 (한 문장)

손으로 4×4 부분 마방진을 풀 때 **행 합 34만**으로 빈칸 후보를 종이에 적어 좁히는 데 **약 24분**을 쓰고, **열·대각선은 넣어 본 뒤에야** 확인하며 **왜 틀렸는지 보기보다** 빈칸 숫자를 **맞을 때까지 바꿔 넣는** 방식으로 **약 25분**을 쓴다.

### Mom Test 증거 3줄

1. “**행 합 34 우선** 기준을 잡았어” / “**넣어본 뒤에**” (열·대각선 확인)
2. “**맞을 때까지 바꿔본 것**이지, 안 되는 것을 먼저 찾기보다는” / “**5번** 정도 시도”
3. “**후보군 찾기**에 시간을 보냈지” / “**적어내며** 했었지” / “후보는 **1개밖에 못 찾았어**”

### Command 계약: `validate_lines` (Session 3 SSOT)

| 구분 | 내용 |
|------|------|
| **시그니처** | `validate_lines(grid) -> dict` |
| **입력** | `grid` = 4×4 `list[list[int]]`, 각 값 **0~16** (`0`=빈칸) |
| **출력** | `{ "status": "pass" \| "fail" \| "incomplete", "failed_lines": [줄ID…] }` |
| **incomplete** | 빈칸(`0`)이 하나라도 있으면 → `status=incomplete`, `failed_lines=[]` (10선 판정 **보류**) |
| **pass** | 16칸 완성 + **10선 모두 34** → `status=pass`, `failed_lines=[]` |
| **fail** | 16칸 완성인데 합≠34인 줄이 있으면 → `status=fail`, 해당 줄 ID를 `failed_lines`에 |
| **줄 ID** | 행 **R1~R4** · 열 **C1~C4** · 대각 **D1**(주) / **D2**(반) |

> **범위 밖:** 중복·1~16 밖 값(17 등)은 `validate_lines`가 다루지 않음 → Rule **R2** · Test **D-003**.

### Test Loop 시나리오 (D-001~D-003)

| ID | 시나리오 | 대상 | 기대 |
|----|----------|------|------|
| **D-001** | 표준 4×4 완전 마방진 (1~16, `0` 없음) | `validate_lines` | `status=pass`, `failed_lines=[]` |
| **D-002** | **4행 모두 합 34**이나 **한 열 또는 대각선 ≠ 34** (1월 “반대로 넣었더니”) | `validate_lines` | `status=fail`, `failed_lines`에 **C*** 또는 **D1/D2** |
| **D-003** | 숫자 **중복** 또는 **1~16 밖** 값 (16칸 완성 여부 무관) | Rule **R2** (선행 검사) | 무결성 **fail** — `validate_lines` **미호출** 또는 결과와 **별도** 판정 |
